## 二、功能测试 (手动执行)

### 1. 基本生成测试
- [ ] 访问 http://localhost:5000 成功
- [ ] 填写需求和示例用例，点击生成，结果正常显示
- [ ] 生成结果有 Markdown 和 CSV 格式
- [ ] 点击"下载 Markdown"和"下载 CSV"，下载文件能打开

### 2. 会话持久化测试
- [ ] 生成用例后，URL 自动变为 `?session=xxx`
- [ ] 刷新页面，用例和标签状态不丢失
- [ ] 重新打开浏览器，粘贴 URL，能正确加载历史会话

### 3. 用例标签测试
- [ ] 点击某条用例的"✅可投产"，标签变高亮
- [ ] 点击"❌舍弃"和"⚠️待修改"，标签切换正常
- [ ] 点击"全标 ✅可投产"，所有用例标签改变
- [ ] 点击"全标 ❌舍弃"和"全标 ⚠️待修改"，批量操作正常

### 4. 用例编辑测试
- [ ] 点击"✏️编辑"，弹出编辑框
- [ ] 修改某条用例的字段，点击保存，卡片内容更新
- [ ] 卡片上显示"✏️已编辑"标记
- [ ] 刷新页面，编辑结果不丢失

### 5. 多轮对话反馈测试
- [ ] 点击"💬反馈"，弹出反馈框
- [ ] 输入反馈内容，点击提交，触发单条用例修改
- [ ] 该条用例的内容按照反馈更新
- [ ] 其他用例保持不变

### 6. 规则学习测试
- [ ] 给用例打标签后，点击"📚学习规则"
- [ ] 弹出提示"学习完成！共获得 N 条规则"
- [ ] 再次生成新用例，规则被自动应用
- [ ] 生成结果质量明显提升（有改进）

### 7. 下载选中测试
- [ ] 只下载标为"✅可投产"的用例
- [ ] 下载的 CSV/Markdown 只包含选中用例
- [ ] 文件名正确，内容完整
