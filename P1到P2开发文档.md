## P2 新增功能（详细）

### 1. 会话持久化系统

#### 实现思路
- 之前：会话只在内存，刷新页面丢失
- 现在：会话保存到 `data/memory/sessions/{session_id}.json`
- URL 自动带 `?session=xxx`，刷新不丢失

#### 新增文件
- `src/memory/memory_store.py` — Memory 存储管理
- 修改 `web/app.py` 支持会话加载/保存

#### 新增 API
- `/api/session/load` — 加载历史会话
- `/api/session/labels` — 保存用例标签状态

---

### 2. 用例标签系统

#### 功能说明
- ✅ 可投产 — 标记高质量用例
- ❌ 舍弃 — 标记低质量用例
- ⚠️ 待修改 — 标记需要改进的用例

#### 实现细节
- 标签状态即时保存到会话
- 批量操作支持（全标可投产/舍弃/待修改）
- 界面显示高亮标签按钮

---

### 3. 用例编辑功能

#### 功能说明
- 支持手动编辑任意用例的任意字段
- 保存编辑前后对照
- 显示"✏️已编辑"标记

#### 新增界面
- 编辑按钮
- 编辑弹窗，可修改所有字段
- 编辑历史存储到 `__history`

---

### 4. 多轮对话反馈

#### 功能说明
- 用户输入反馈内容
- AI 只修改该条用例
- 其他用例完全不变

#### 新增 API
- `/api/generate-single` — 只修改单条用例

---

### 5. 规则学习系统

#### 核心设计
借鉴 OpenClaw 的记忆与学习机制：

1. **规则提取**
   - 用户标记用例后，点击"📚学习规则"
   - AI 自动从标签和反馈中提取规则
   - 规则保存到 `data/memory/rules.json`

2. **规则注入**
   - 下次生成用例时
   - 自动将已学规则注入到 system_prompt
   - 提升生成质量

#### 新增文件
- `src/memory/rule_extractor.py` — 规则提取器
- `src/memory/rule_injector.py` — 规则注入器
- 修改 `src/workflow/pipeline.py` 支持规则注入

#### 新增 API
- `/api/extract-rules` — 触发规则学习

---

### 6. 导出功能增强

#### 功能说明
- "下载选中用例" → "下载可投产用例"
- 只下载标为"✅可投产"的用例
- 文件名规范，方便管理

## 依赖说明

**零第三方依赖！**

**全是 Python 标准库**：
- `http.server` — Web 服务
- `urllib` — HTTP 请求
- `json` — JSON 解析
- `csv` — CSV 导出
- `difflib` — 字符串相似度
- `uuid` — 会话 ID 生成
- `pathlib` — 文件路径
- `socketserver` — 多线程 Web 服务
