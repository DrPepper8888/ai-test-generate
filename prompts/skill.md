# 测试规则模板

你可以在这里定义项目专属的测试用例编写规则。

## 使用方式

1. 编辑本文件，添加你的规则
2. 保存后，AI 生成时会自动遵守这些规则
3. 支持多行规则，每行一条

## 示例规则

```markdown
# 测试用例必须包含以下字段
1. ID：用例编号
2. 标题：用例名称
3. 前置条件：必要的初始状态
4. 测试步骤：清晰的执行步骤
5. 预期结果：明确的可验证结果
6. 优先级：P0/P1/P2

# 命名规范
- ID 格式：TC_XXX（如 TC_001）
- 优先级定义：
  - P0：核心功能，阻塞性缺陷
  - P1：重要功能，非阻塞性缺陷
  - P2：边缘场景，非功能性问题

# 描述规范
- 测试步骤使用动词开头（如：点击、输入、验证）
- 预期结果描述可观测的变化（如：页面跳转、数据显示）
- 避免模糊描述（如：正常、成功）
```

## 提示

- 使用 Markdown 格式
- 以 `#` 开头的行是注释，不会被注入
- 保留 `---` 分隔线之前的注释说明
