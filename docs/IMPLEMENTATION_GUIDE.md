## 2. 前端与后端如何通信

### 2.1 通信方式：HTTP REST API

前端和后端通过 **HTTP 请求** 进行通信，就像浏览器访问网页一样。

```
前端 ──HTTP请求──► 后端
     ◄─JSON响应──
```

### 2.2 请求示例

**生成用例请求：**
```javascript
// 前端代码 (index.html)
fetch('/api/generate', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    requirement: '用户登录功能',
    example: '[{"id": "TC_001", ...}]',
    count: 7
  })
})
```

**后端响应：**
```json
{
  "success": true,
  "cases": [
    {"id": "TC_002", "title": "正常登录", ...},
    {"id": "TC_003", "title": "密码错误", ...}
  ],
  "markdown": "# 测试用例\n\n## TC_002...",
  "csv": "ID,标题,..."
}
```

### 2.3 前后端文件对应关系

| 前端功能 | 前端文件 | 后端文件 |
|---------|---------|---------|
| 页面展示 | `web/templates/index.html` | - |
| 请求发送 | JavaScript 代码 | - |
| 接口处理 | - | `web/app.py` |
| 用例生成 | - | `src/workflow/pipeline.py` |
| AI 调用 | - | `src/api/llm_client.py` |

## 5. 核心模块说明

### 5.1 前端模块 (index.html)

#### 总指令框 (Chat Box)
- **位置**：页面左侧最上方
- **作用**：接收用户的宏观调控指令
- **存储**：保存在浏览器 localStorage
- **传递**：生成时自动追加到需求描述

```javascript
// 用户输入指令
"注意时间范围要完整覆盖11:30-13:00"
    ↓
// 生成时追加到需求
requirement + "\n\n【用户指令】\n注意时间范围要完整..."
    ↓
// 发送给后端
```

#### Harness 进化面板
- **位置**：页面左侧底部
- **作用**：展示已学习的规则
- **功能**：
  - 显示规则数量
  - 预览最近学习的规则
  - 点击「📋 管理」打开规则管理弹窗

#### 用例操作区
- **位置**：页面右侧结果区域
- **功能**：
  - 显示生成的用例表格
  - 每条用例的操作按钮
  - 批量操作（全部通过、选择删除）

### 5.2 后端模块

#### app.py - HTTP 服务器

| 路由 | 方法 | 功能 |
|------|------|------|
| `/` | GET | 返回前端页面 |
| `/api/generate` | POST | 生成测试用例 |
| `/api/feedback` | POST | 提交用户反馈 |
| `/api/rules/learn` | POST | 自动学习规则 |
| `/api/rules` | GET | 获取规则列表 |
| `/api/health` | GET | 健康检查 |

#### pipeline.py - 用例生成流程

```
┌─────────────────────────────────────────────────────────────┐
│                    Pipeline 用例生成流程                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 加载提示词模板                                           │

│  2. 组装用户消息                                            │
│     └── 需求 + 示例 + 指令 + 历史规则                         │
```

## 7. 常见问题 FAQ

### Q1: 为什么有时候生成会失败？

**原因可能有：**
1. AI API 调用超时（网络问题）
2. 返回的 JSON 格式不正确（AI 输出问题）
3. 请求内容过长超出限制

**解决方案：**
- 检查 LLM 配置是否正确
- 简化需求描述
- 减少生成数量

### Q2: 规则是如何学习的？

**学习流程：**

```
用户给反馈（👍/👎）
         │
         ▼
┌─────────────────┐
│  分析反馈内容    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  AI 自审规则    │  ← 检查规则是否有效
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  规则入库       │  ← 保存到 rules.json
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  下次生成应用   │  ← 自动注入到提示词
└─────────────────┘
```

### Q3: 数据存储在哪里？

| 数据类型 | 存储位置 | 说明 |
|---------|---------|------|
| 规则库 | `data/memory/rules.json` | 持久化存储 |
| 生成历史 | `data/history.json` | 持久化存储 |
| 聊天历史 | 浏览器 localStorage | 仅本地保存 |
| 用户偏好 | 浏览器 localStorage | 仅本地保存 |

### Q4: 如何导出用例？

**方式一：下载文件**
- 点击「📥 Markdown」下载 .md 文件
- 点击「📥 CSV」下载 .csv 文件
- 点击「📥 JSON」下载 .json 文件

**方式二：复制**
- 切换到「Markdown」或「JSON」标签页
- 点击「📋 复制」按钮

### Q5: 如何清空已学习的规则？

1. 点击 Harness 面板的「📋 管理」按钮
2. 打开规则管理弹窗
3. 点击「🧹 清理已淘汰」删除无效规则
4. 如需删除所有规则，可手动编辑 `data/memory/rules.json`

## 8. 本地模型使用指南

### 8.1 本地模型特点

使用本地模型（如 Ollama）有以下特点：

| 特点 | 说明 |
|------|------|
| ✅ 隐私安全 | 数据不离开本地 |
| ✅ 免费使用 | 无 API 调用费用 |
| ⚠️ 性能有限 | 生成速度较慢 |
| ⚠️ 上下文有限 | 输入输出 token 有限制 |
| ⚠️ 输出可能截断 | 过长输出会被截断 |

### 8.2 本地模型限制

**常见错误及解决方案：**

| 错误类型 | 原因 | 解决方案 |
|---------|------|---------|
| 🤖 本地模型限制 | 输出被截断 | 减少生成数量（5条） |
| 🐢 本地模型较慢 | 模型响应慢 | 耐心等待或增加 timeout |
| 📋 JSON 格式错误 | 输出太长 | 简化需求描述 |

### 8.3 优化建议

**使用本地模型时：**

1. **减少生成数量**
   - 建议：5-7 条
   - 原因：减少输出长度，降低截断风险

2. **简化需求描述**
   - 建议：200 字以内
   - 原因：节省上下文空间给输出

3. **增加 timeout 配置**
   ```json
   {
     "llm": {
       "timeout": 180
     }
   }
   ```

4. **使用精简的提示词**
   - 避免过长的示例用例
   - 直接描述核心需求

### 8.4 推荐的本地模型配置

```json
{
  "llm": {
    "base_url": "http://localhost:11434",
    "model_name": "qwen2.5:7b",
    "timeout": 180,
    "max_tokens": 4096
  },
  "generation": {
    "default_count": 5,
    "max_count": 10
  }
}
```

**推荐模型：**
- `qwen2.5:7b` - 平衡性能和速度
- `qwen2.5:14b` - 性能更好，需要更多内存
- `llama3:8b` - 开源选择
