> 版本：P1.0 | 状态：已完成

---

## 1. 项目概述

**项目目标**：测试工程师只需提供一条示例用例 + 需求描述，AI 自动批量生成测试用例。

**核心能力**：
- 支持文本输入和文件上传（Excel/CSV）
- 自动去重（相似度 > 0.98）
- 双 Agent 评审（可选）
- 历史记录查询
- 用户反馈收集

## 3. 核心模块说明

### 3.1 Web 服务 —— `web/app.py`

使用 Python 标准库 `http.server`，零第三方 Web 依赖。

**启动方式**：
```bash
python web/app.py
```

**API 接口**：

| 接口 | 方法 | 参数 | 说明 |
|------|------|------|------|
| `/` | GET | - | 返回 Web 页面 |
| `/api/generate` | POST | requirement, example, count, review_mode | 生成用例 |
| `/api/import` | POST | file | 导入 Excel/CSV |
| `/api/history` | GET | X-History-Limit, X-History-Offset | 查询历史 |
| `/api/history/detail` | POST | id | 获取历史详情 |
| `/api/feedback` | POST | rating, type, comment, cases_count | 提交反馈 |
| `/api/config` | GET/POST | base_url, model_name, api_key | 配置管理 |
| `/api/health` | GET | - | LLM 健康检查 |

**文件上传格式**：
```javascript
// 前端使用 FormData 上传
const formData = new FormData();
formData.append('file', fileElement.files[0]);
const resp = await fetch('/api/import', { method: 'POST', body: formData });
```

### 3.2 LLM 客户端 —— `src/api/llm_client.py`

统一 HTTP 客户端，支持多供应商。

**使用方式**：
```python
from src.api.llm_client import LLMClient

client = LLMClient(
    base_url="https://ark.cn-beijing.volces.com/api/coding",
    model_name="deepseek-v3.2",
    api_key="ark-xxxxx"
)

response = client.chat(system_prompt, user_message)
```

**供应商自动识别**：
- URL 包含 `anthropic.com` → Anthropic 协议
- URL 包含 `azure.com` → Azure 协议
- URL 包含 `ark.cn-beijing.volces.com/api/coding` → 火山方舟 Coding Plan
- 其他 → OpenAI 兼容协议

### 3.3 状态机编排 —— `src/workflow/pipeline.py`

**核心流程**：

```
run(requirement, example, count, review_mode=False)
  ├── 1. 加载 system_prompt
  ├── 2. 提取示例字段结构
  ├── 3. 分批（基于 token 估算）
  ├── 4. 调用 LLM（含重试）
  ├── 5. 去重（阈值 > 0.98）
  ├── 6. 评审（可选，review_mode=True）
  ├── 7. 导出 Markdown/CSV
  └── 8. 保存历史
```

**返回结构**：
```python
{
    "success": True,
    "cases": [...],
    "markdown": "...",
    "csv": "...",
    "error": None,
    "state": "done",
    "retries": 0,
    "count": 7,
    "deduplicated": 0,        # P1 新增
    "review": {               # P1 新增，仅 review_mode=True 时
        "score": 8,
        "issues": [],
        "suggestions": []
    }
}
```

### 3.4 去重工具 —— `src/tools/deduplicator.py`

使用 Python 标准库 `difflib` 计算字符串相似度。

**核心函数**：
```python
from src.tools.deduplicator import deduplicate, similarity

# 计算相似度（0-1）
sim = similarity("测试用例A", "测试用例A")

# 去重（阈值 0.98）
result = deduplicate(cases, threshold=0.98)
```

**算法说明**：
- 将用例转换为 `k1=v1 | k2=v2` 格式
- 使用 `difflib.SequenceMatcher` 计算相似度
- 阈值 > 0.98 则去除

### 3.5 文件导入 —— `src/tools/importer.py`

支持 .xlsx, .xls, .csv 格式。

```python
from src.tools.importer import import_file_to_cases

cases = import_file_to_cases("test.xlsx")
# 返回: [{"id": "TC_001", "title": "..."}, ...]
```

### 3.6 评审机制

**实现方式**：

1. 创建 `prompts/review_prompt.md` 评审提示词
2. 在 `pipeline.py` 中调用第二次 LLM
3. 解析评审结果 JSON

**使用**：
```javascript
// 开启评审
fetch('/api/generate', {
    method: 'POST',
    body: JSON.stringify({
        requirement: '...',
        example: '[...]',
        count: 7,
        review_mode: true
    })
})
```

## 7. 常见问题

### Q: 如何切换不同 LLM？
A: 修改 `config.json` 中的 `base_url` 和 `model_name`。

### Q: 如何开启评审？
A: 调用 API 时设置 `review_mode: true`。

### Q: 去重阈值是多少？
A: 固定 0.98（相似度超过 98% 去重）。

### Q: 支持哪些文件格式？
A: .xlsx, .xls, .csv（需要 openpyxl）。

### Q: 如何查看历史记录？
A: 访问 `/api/history` 或 `/api/history/detail`。

## 8. 开发规范

### 8.1 代码风格
- 使用 Python 标准库，减少第三方依赖
- 类型提示使用内置 `typing`
- 私有方法以 `_` 开头

### 8.2 错误处理
- LLM 调用失败自动重试（最多 `max_retries` 次）
- 文件保存失败不影响主流程
- 错误信息返回给前端展示

### 8.3 测试要求
- 启动前验证 import 正常
- 检查 config.json 格式正确
