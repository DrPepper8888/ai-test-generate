# 【格式约束】

⚠️ 这是最重要的约束，违反将导致输出被拒绝：

1. **字段完全一致**：输出的每条用例必须包含与示例用例完全相同的字段，字段名称、数量、顺序一律不得改变
2. **字段内容格式一致**：如果示例的某个字段（如 `steps`、`测试步骤`）包含多个编号步骤（形如 `1. xxx 2. xxx` 或数组格式），生成的用例的对应字段也必须保持**多步骤格式**，绝对不能只写第1步！
3. **ID 顺序编号**：用例 ID 从 TC_002 开始，依次递增（TC_002、TC_003...）
3. **输出为 JSON 数组**：整体输出必须是合法 JSON 数组，每个元素是一个对象
4. **不重复示例**：不得生成与示例用例（TC_001）完全相同或高度相似的用例
5. **语言一致性**：示例用中文则输出中文，示例用英文则输出英文
6. **场景标记**：每条用例必须包含 `_scenario` 字段，取值：normal / boundary / attack / business

**输出格式示例（假设示例有 id、title、precondition、steps、expected 五个字段）：**
```json
[
  {
    "id": "TC_002",
    "title": "...",
    "precondition": "...",
    "steps": "...",
    "expected": "...",
    "_scenario": "normal"
  }
]
```
