# AI 测试用例生成器 - 功能说明书

> 项目状态：P2 阶段已完成

---

## 项目概述

**目标**：测试工程师只需提供一条示例用例 + 需求描述，AI 自动批量生成测试用例，并支持从用户反馈中学习，越用越准。

---

## 核心功能

### 1. 基本生成功能（P0）

| 功能 | 说明 |
|------|------|
| 文本输入 | 支持文本输入测试需求和示例用例 |
| 文件导入 | 支持 Excel（.xlsx/.xls）和 CSV 文件导入 |
| 示例解析 | 自动解析示例用例的字段结构 |
| 批量生成 | 支持生成 3-20 条用例 |
| 多场景覆盖 | 自动覆盖正常功能、边界值、异常输入、特殊业务四类场景 |
| 双 Agent 评审 | 可选开启二次 LLM 评审，评分和建议 |
| 结果导出 | 支持 Markdown 和 CSV 格式导出 |

---

### 2. 质量优化功能（P1）

| 功能 | 说明 |
|------|------|
| 简单去重 | 基于字符串相似度（阈值>0.98）自动去重 |
| Token 控制 | 自动估算 token 数，超过阈值自动分批生成 |
| 上下文分批 | 避免单次请求 token 超限 |
| 用户反馈收集 | 支持用户提交评分、类型、文字反馈 |
| 历史生成保存 | 保存生成历史，支持查询 |

---

### 3. 记忆与学习功能（P2）

| 功能 | 说明 |
|------|------|
| 会话持久化 | 会话保存到文件，刷新页面不丢失 |
| 用例标签 | 支持给用例打标签：✅可投产 / ❌舍弃 / ⚠️待修改 |
| 批量标签 | 支持全标可投产/全标舍弃/全标待修改 |
| 用例编辑 | 支持手动编辑任意用例的任意字段 |
| 编辑历史 | 保存编辑前后的对照，显示"✏️已编辑"标记 |
| 多轮对话反馈 | 用户输入反馈，AI 只修改该条用例，其他用例不变 |
| 规则学习系统 | 用户标记用例后，AI 自动提取测试规则 |
| 规则库管理 | 学习到的规则保存到 `data/memory/rules.json` |
| 规则自动应用 | 下次生成用例时，自动将已学规则注入到 system prompt |

---

### 4. 导出功能增强（P2）

| 功能 | 说明 |
|------|------|
| 下载可投产用例 | 只下载标为"✅可投产"的用例 |
| 下载全部用例 | 支持下载全部用例（Markdown/CSV） |
| 文件名规范 | 自动命名，方便管理 |

## LLM 支持

### 供应商支持（自动识别）

| 供应商 | 说明 | 认证方式 |
|--------|------|----------|
| OpenAI | 标准 OpenAI API | Bearer Token |
| Anthropic | 标准 Anthropic API | x-api-key |
| 火山方舟 Coding Plan | 兼容 Anthropic API | Bearer Token |
| Azure OpenAI | 兼容 Azure API | api-key |
| Ollama | 内网本地部署 | 无需认证 |
| vLLM / FastChat / 任意 OpenAI 兼容 | 内网部署 | 可选 Bearer Token |

## 配置文件

### config.json

```json
{
  "llm": {
    "base_url": "https://ark.cn-beijing.volces.com/api/coding",
    "model_name": "deepseek-v3.2",
    "api_key": "ark-xxxx",
    "timeout": 120,
    "max_tokens": 4000,
    "temperature": 0.7
  },
  "generation": {
    "default_count": 7,
    "min_count": 3,
    "max_count": 20,
    "batch_size": 20,
    "max_retries": 2
  },
  "storage": {
    "history_file": "data/history.json",
    "feedback_file": "data/feedback.json"
  }
}
```

## 部署说明

### 环境要求
- Python 3.8+
- **无需安装任何第三方包！**

### 启动服务
```bash
python web/app.py
```

### 访问地址
- http://localhost:5000

## 数据存储位置

| 数据类型 | 位置 |
|---------|------|
